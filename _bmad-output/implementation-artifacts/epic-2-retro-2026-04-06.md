## What Could Be Improved

- **Story file hygiene** — Stories 2.3 and 2.4 were not properly finalized (unchecked task boxes, template placeholders in Dev Record). Code was done but story tracking lagged.
- **Deferred items growing** — 6 from Epic 1 + 8 from Epic 2 = 14 total. Most are Phase 2+ but should be tracked centrally.
- **Error handling initially too naive** — TaskRunner's error handler had multiple issues (wrong listener args, no cascading protection, hardcoded strings). The pattern needs to be established before Epic 3.

## Key Technical Decisions Validated

- **Atomic writes via tempfile+os.replace** — works cross-platform, tested
- **Gate Protocol with configurable commands** — no hardcoded tools, `FileNotFoundError` returns structured `GateResult`
- **GateRunner exception safety** — `try/except` around each `gate.run()` maintains no-short-circuit contract even on unexpected exceptions
- **TaskRunner constructor injection** — 9 parameters is a lot but enables complete testability. Integration tests inject mocks at every boundary.
- **Epic-wide code review** — more valuable than per-story reviews for finding cross-component issues
